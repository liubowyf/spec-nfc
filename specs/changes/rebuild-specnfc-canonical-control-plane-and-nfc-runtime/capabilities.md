# Capabilities

## 新增能力
1. **canonical control plane 定义**：把 `.specnfc/` 从模板输出目录提升为正式协议源。
2. **中文 `nfc` 运行时模型**：把深访、规划、日志、handoff、writeback 收编为 `specnfc` 自有运行时。
3. **多层对象与索引**：补齐 team/project/repo/change/integration 多层 schema 与 index 关系。
4. **治理模式模型**：给出 advisory / guided / strict / locked 四档治理与 gate 差异。
5. **统一 next-step protocol**：把状态建议从散点输出升级成结构化 contract。

## 增强能力
1. **init**：从初始化目录升级为项目协议接入动作。
2. **status**：从状态摘要升级为 control-plane dashboard。
3. **doctor**：从健康检查升级为协议一致性检查器。
4. **upgrade**：从模板刷新升级为协议迁移器。
5. **change / integration**：从目录管理升级为 canonical work object manager / contract alignment manager。

## 保留能力
- change dossier 与 integration dossier 的基本对象域
- 项目记忆、readingPath、入口索引、规则快照等既有成果
- 发布安装与 manifest 校验闭环
- managed files、diff 预览、保守升级策略
